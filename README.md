## 프로젝트 개요

이 프로젝트는 ADeX 플랫폼에서 수집된 광고 효과 데이터를 시각화하고 분석 결과를 보고서 형태로 제공하는 템플릿입니다. 

- **총 17페이지** 분량의 상세한 효과 분석 보고서
- **Chart.js**를 활용한 다양한 데이터 시각화
- **인쇄 최적화** 레이아웃 (A4 용지 기준)
- **템플릿 변수**를 통한 동적 콘텐츠 생성

## 주요 기능

✅ **다양한 차트 시각화**
- 미디어그룹별 유동/노출인구 비교 차트
- 일별/요일별/시간대별 노출인구 추이 차트
- 연령・성별 분포 피라미드 차트

✅ **상세한 성과 분석**
- 미디어그룹별 성과 비교 테이블
- 주요 오디언스 분석
- 광고 송출 분석 (일반/싱크/타임)

✅ **인사이트 제공**
- 자동 인사이트 생성
- 최다 노출인구 날짜/요일/시간대 표시
- 유사 미디어그룹 추천

## 기술 스택

- **HTML5** - 마크업
- **CSS3** - 스타일링 (인쇄 최적화)
- **JavaScript (ES6+)** - 동적 기능 및 차트 생성
- **Chart.js 4.4.0** - 데이터 시각화 라이브러리
- **Pretendard 폰트** - 웹 폰트 (CDN)

## 빠른 시작

### 1. 프로젝트 다운로드

```bash
# 프로젝트 디렉토리로 이동
cd adex-report-project
```

### 2. 로컬 서버 실행

#### Python 사용
```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

#### Node.js 사용
```bash
# http-server 설치
npm install -g http-server

# 서버 실행
http-server -p 8000
```

#### PHP 사용
```bash
php -S localhost:8000
```

### 3. 브라우저에서 확인

브라우저에서 `http://localhost:8000` 접속

### 4. 템플릿 변수 치환

서버 측에서 템플릿 엔진을 사용하여 `{{변수명}}` 형식의 변수를 실제 데이터로 치환합니다.

**Handlebars 예시:**
```handlebars
<h2>{{광고계약명}}</h2>
<span>{{광고시작일}} ~ {{광고종료일}}</span>
```

**Jinja2 예시:**
```jinja2
<h2>{{ 광고계약명 }}</h2>
<span>{{ 광고시작일 }} ~ {{ 광고종료일 }}</span>
```

## 페이지별 데이터 입력 가이드 (Figma 주석 기반)

### 1. 표지 (`표지`)
- **필수 변수**: `{{광고계약명}}`, `{{광고주명}}`, `{{광고시작일}}`, `{{광고종료일}}`, `{{운영일수}}`, `{{보고서발행일}}`.
- **운영일 계산**: 시작일~종료일 기준 일수를 반올림 없이 정수로 표기.
- **표시 규칙**: 표지에는 페이지 번호를 노출하지 않으며, 반투명 배경 위 텍스트 대비를 유지합니다.

### 2. 목차 (`목차`)
- **미디어그룹 정렬**: 유동 대비 노출 비율이 높은 순으로 정렬하되, 동일 비율일 경우 ㄱ-ㅎ → A-Z 순으로 정렬합니다.
- **페이지 넘버링**: 표지를 제외하고 `section.page`가 렌더링될 때마다 `counter-increment: page-number`가 적용되어 목차부터 번호를 시작합니다.
- **간지 처리**: 간지 페이지는 카운트에는 포함되지만 페이지 번호 요소는 숨깁니다(`.page-divider .page-number`).

### 3. 간지 1 (`간지 1`)
- 섹션 구분 이미지를 사용하며 추가 데이터 입력은 없습니다.
- 페이지 번호 요소는 DOM에 존재하되 시각적으로 숨겨 CSS 카운터 정상 동작을 유지합니다.

### 4. 미디어그룹 상품 정보 (`미디어그룹 상품 정보`)
- **기본 필드**: `{{광고계약명}}`, `{{광고주명}}`, `{{미디어그룹명_N}}`, `{{리포트발행일}}`.
- **디스플레이 정보**: LCD/LED 유형별 인치, 가로/세로 사이즈, pitch, 대수 등 동일 스펙 장비를 그룹화하여 입력합니다.
- **기간 정보**: `{{광고시작일}}`, `{{광고종료일}}`, `{{총광고일수}}`.
- **표기 규칙**: 유형별로 행을 추가하며, 값이 없을 경우 0이나 빈 칸 대신 대시(`-`)를 사용해 공란을 피합니다.

### 5. 주요 광고 성과 - Ⅰ (`주요 광고 성과 - Ⅰ`)
- **핵심 지표**: 총 유동 인구, 총 노출 인구, 유동 대비 노출 비율(`노출 인구 ÷ 유동 인구 × 100`), 광고 효과 수준.
- **기간 구간**: 기본 구간 외에 하단 비교 구간의 시작/종료일도 `YYYY.MM.DD` 형식으로 제공합니다.
- **서식**: 비율은 소수점 첫째 자리까지 반올림하고 `%` 없이 정수+소수 형태(예: `78.5`)로 표기합니다.

### 6. 주요 광고 성과 2 (`주요 광고 성과 2`)
- **주요 오디언스**: 노출 인구에서 비중이 높은 성별/연령대 Top3를 입력합니다.
- **기간 정보**: 상단 시계열 구간의 시작/종료일을 동일 형식으로 노출합니다.
- **서식 규칙**: Top1 항목은 강조(굵게) 처리하고, 비율 데이터는 소수점 첫째 자리까지 표시합니다.

### 7. 주요 광고 성과 3 (`주요 광고 성과 3`)
- **테이블 데이터**:
  - 미디어그룹 리스트는 유동 대비 노출 비율 내림차순(동률 시 ㄱㄴㄷ → A-Z).
  - 각 그룹의 총 유동 인구, 총 노출 인구, 노출 비율(`노출÷유동×100`), CPM(`최종 광고비 ÷ 노출 인구 × 1,000`)을 제공합니다.
- **강조 규칙**:
  - CPM을 제외한 각 열에서 가장 높은 값을 `font-weight: 700`으로 표시합니다.
  - 가장 높은 노출 인구 그룹을 추가적으로 강조(예: 배경 컬러 혹은 라벨).
- **유사 미디어그룹**: 운영 광고와 동일한 성별/연령대 오디언스를 갖는 추천 그룹의 CPM을 산출해 비교 영역에 입력합니다.

### 8. Frame 108 (`연령/성별 인구 피라미드`)
- **입력 데이터**: 미디어그룹별 성별-연령대별 노출 인구 비율/인구수.
- **강조 규칙**: 남성/여성 각각 가장 높은 비율을 차트 내에서 강조하고 해당 인구수를 라벨로 출력합니다.

### 9. Frame 109 (`일별 노출 비교`)
- **차트 데이터**: 일자 배열, 미디어그룹별 일별 노출 인구 누적 값.
- **라벨**:
  - 전체 미디어그룹 합계가 가장 높은 날짜의 총 노출 인구를 막대 상단에 표기합니다.
  - 각 미디어그룹의 피크 값은 해당 스택 영역에 직접 라벨링합니다.
- **보조선**: 전체 미디어그룹 일평균 노출 인구를 산출해 Chart.js 보조선으로 노출합니다.

### 10. 주요 광고 성과 4-3 (`요일별 노출 비교`)
- **차트 데이터**: 요일별 일 평균 노출 인구 (`해당 요일 총합 ÷ 데이터 존재 일수`).
- **강조 규칙**: 총합이 가장 큰 요일의 막대 상단에 값을 표시하고, 각 미디어그룹 피크 요일에도 라벨을 추가합니다.
- **보조선**: 전체 미디어그룹의 요일별 일 평균을 꺾은선 형태로 중첩합니다.

### 11. Frame 110 (`시간대별 노출 비교`)
- **차트 데이터**: 시간대별 평균 노출 인구, 미디어그룹별 값, 전체 평균.
- **강조 규칙**: 누적 합계가 가장 높은 시간대를 강조하여 총 인구수를 표시하고, 각 그룹의 최대 시간대에 명시적 라벨을 추가합니다.
- **보조선**: 전체 평균(혹은 목표선)을 보조선으로 표시해 추세 대비 비교가 가능하도록 합니다.

### 12. Frame 103 (`미디어그룹 리스트 요약`)
- **콘텐츠**: 미디어그룹 이름 리스트, 섹션 2 진입 안내 텍스트.
- **정렬 규칙**: 앞선 페이지와 동일한 정렬 기준(유동 대비 노출 비율, 동률 시 가나다/알파벳).

### 13~16. 미디어그룹별 상세 세트
- **생성 규칙**: 보고 대상 미디어그룹마다 4페이지 묶음을 복제하여 13~16페이지 범위를 순차적으로 확장합니다. 예를 들어 3개의 미디어그룹이면 총 12페이지(3×4)가 추가되며, 이후 끝 표지가 따라옵니다.
- **페이지 구성**: `미디어그룹 정보` → `노출/송출 분석` → `노출 효과 시각화 1` → `노출 효과 시각화 2` 순으로 4페이지가 한 묶음이며 미디어그룹 수만큼 반복합니다.
- **공통 필수 변수**: `{{미디어그룹명}}`, `{{광고시작일}}`, `{{광고종료일}}`, `{{운영일수}}`, 주소, 지도 이미지, 운영 현장 사진 2장(없을 경우 상품 이미지).
- **미디어 스펙**: 디스플레이 유형, 사이즈, pitch, 운영 시간, 콘텐츠 재생 정보, 광고 운영 방식(일반형/싱크형 등)을 모두 기입합니다.
- **노출 효과 분석**:
  - 유동/노출 인구 누적, 노출 비율, CPM, 주요 오디언스, 주요 요일/시간대를 Figma 주석 기준으로 계산합니다.
  - `총 송출 횟수`, `일 평균 송출 횟수`, `일 최소 송출 보장수`, `송출 보장 달성률`, `평균 송출 주기`를 CMS 또는 미디어사 데이터로 채웁니다.
  - 타임 광고를 운영한 경우에만 타임 섹션(총 노출 시간/보장 시간, 달성률)을 표시합니다.
- **시각화 1**: 연령/성별 인구 피라미드, 노출 비중 최고 연령·성별 강조, 노출 인구 최다 일자(`YYYY.MM.DD`) 표시.
- **시각화 2**: 요일별/시간대별 차트에서 남녀 구분 누적 그래프, 최고 요일/시간대 라벨, 전체 평균 보조선 규칙을 준수합니다.

### 17. 끝 표지 (`끝 표지`)
- 문의처, 저작권 문구를 고정 텍스트로 노출합니다.
- 페이지 번호는 숨기되 PDF 페이지 순서는 유지합니다.

## 페이지 전환 및 넘버링 가이드

- **페이지 카운터 초기화**: `body { counter-reset: page-number; }`로 초기화되며, `section.page:not(.page-cover)`에서 카운터가 증가합니다. 표지에는 페이지 번호가 표시되지 않습니다.
- **간지 페이지 처리**: 간지(예: `간지 1`, 섹션 구분 페이지)는 카운터 증가 대상이지만 `.page-divider` 클래스에서 페이지 번호 요소를 숨겨 시각적 넘버링을 건너뜁니다.
- **반복 섹션 페이지 분리**: `.media-info-page-break` 또는 `section.page` 자체에 정의된 `page-break-before/after`를 활용해 미디어그룹 단위 페이지가 정확히 끊어지도록 합니다.
- **마지막 페이지 예외**: `section.page:last-child`는 `page-break-after: auto`가 적용되어 PDF 출력 시 불필요한 공백 페이지가 생성되지 않습니다.
- **동적 페이지 생성 시 주의**:
  - 서버/빌드 단계에서 미디어그룹 개수만큼 13~16페이지 묶음을 렌더링하고, 각 페이지의 `data-media-group-key`를 고유하게 지정합니다.
  - 템플릿 복제 시 페이지 번호 카운터는 자동 증가하므로 별도 수동 조정이 필요하지 않지만, 간지 및 끝 페이지의 페이지 번호 숨김 상태는 유지해야 합니다.
- **내비게이션 품질 검증**: 브라우저 `print preview`로 페이지 경계를 확인하고, 목차 링크/스크롤 네비게이션을 제공할 경우 `id` 앵커를 목차 순서와 동일하게 배치합니다.

## 개발 가이드

### 템플릿 변수 치환

모든 `{{변수명}}` 형식의 템플릿 변수는 서버 측에서 실제 데이터로 치환되어야 합니다.

**주의사항:**
- 템플릿 변수는 대소문자를 구분합니다
- 날짜는 `YYYY.MM.DD` 형식으로 전달합니다 (예: `2025.01.31`)
- 숫자는 `1,000`과 같이 천 단위마다 콤마를 포함해 전달합니다
- 비율은 소수 둘째 자리에서 반올림하여 소수 첫째 자리까지(`Nn.n`) 표기합니다 (예: `85.4%`)

### JavaScript 데이터 교체

코드 내에 "임시 데이터 (서버에서 위의 템플릿으로 치환)" 주석이 있는 부분은 실제 데이터로 교체해야 합니다.

**예시:**
```javascript
// ❌ 임시 데이터 (교체 필요)
const 일별노출인구데이터 = {
  날짜리스트: ['2024-01-01', '2024-01-02'],
  미디어그룹리스트: [...]
};

// ✅ 서버에서 제공된 실제 데이터로 교체
const 일별노출인구데이터 = 서버에서받은데이터;
```

### 반복문 사용

미디어그룹 리스트 등은 반복문을 사용하여 동적으로 생성합니다.

**Jinja2:**

```jinja2
    <td>{{ item.노출인구 }}명</td>

```

### 미디어그룹 섹션 자동화

`adex-report.html`의 미디어그룹별 상세 페이지(13~16페이지)는 하나의 템플릿으로만 제공됩니다.
보고서 생성 시 반복문을 사용해 미디어그룹 수만큼 템플릿을 복제해 주세요.

- **데이터 구조 예시**
  ```json
  [
    {
      "번호": 1,
      "식별자": "a",
      "이름": "서울광장 미디어타워",
      "광고물유형": "LED",
      "장소적특징": "시청역 사거리",
      "미디어그룹지도이미지": "https://...",
      "미디어그룹번호": 1,
      "미디어위치": "서울특별시 중구",
      "...": "..."
    }
  ]
  ```
- **서버 사이드 반복 렌더링 (Handlebars 예시)**
  ```handlebars
  {{#each 미디어그룹리스트}}
    {{> mediaGroupSection this}}
  {{/each}}
  ```
- **클라이언트 렌더링 예시**
  ```html
  <template id="media-group-section-template">
    <!-- 페이지 13~16 템플릿 -->
  </template>
  <script>
    document.addEventListener('DOMContentLoaded', () => {
      const wrapper = document.querySelector('.page-wrapper');
      const template = document.getElementById('media-group-section-template');

      (window.mediaGroups || []).forEach((group, index) => {
        const clone = template.content.cloneNode(true);
        clone.querySelectorAll('[data-bind]').forEach((el) => {
          const key = el.getAttribute('data-bind');
          if (key in group) el.textContent = group[key];
        });
        clone.querySelectorAll('[data-media-group-key]').forEach((el) => {
          el.setAttribute('data-media-group-key', group.번호 ?? group.식별자 ?? index);
        });
        wrapper.appendChild(clone);
      });
    });
  </script>
  ```
- 페이지 번호, 차트 ID, `data-media-group-key` 등 고유 값은 반복문 안에서 인덱스/식별자를 활용해 동적으로 지정합니다.

- **배경 그래픽**: 인쇄 (배경 이미지 포함)

## 문제 해결

### 차트가 표시되지 않는 경우

1. Chart.js 라이브러리가 로드되었는지 확인
2. 브라우저 콘솔에서 JavaScript 오류 확인
3. 데이터 형식이 올바른지 확인

### 템플릿 변수가 치환되지 않는 경우

1. 템플릿 엔진이 올바르게 설정되었는지 확인
2. 변수명의 대소문자 확인
3. 변수명에 공백이나 특수문자가 없는지 확인

### 스타일이 적용되지 않는 경우

1. `styles.css` 파일이 올바르게 로드되었는지 확인
2. 브라우저 개발자 도구에서 CSS 로드 확인
3. 캐시 문제일 수 있으므로 강력 새로고침 (`Ctrl+Shift+R`)

### 인쇄 시 레이아웃이 깨지는 경우

1. 브라우저 인쇄 설정에서 배경 그래픽 인쇄 옵션 확인
2. 용지 크기가 A4로 설정되었는지 확인
3. 페이지 나누기가 올바른지 확인
